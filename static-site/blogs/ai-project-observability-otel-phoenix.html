How to Setup Your AI Project for Observability Using OTel and Arize Phoenix

Setting up observability for your AI project is like installing a "flight recorder" (OpenTelemetry) and a "mission control dashboard" (Arize Phoenix). This guide walks you through the full setup.

The Big Picture Flow

Before diving into code, let's understand the architecture. Think of it as a pipeline with four components:

  1. The Application: Your code (React, Python, FastAPI, etc.)
  2. The Instrument (OTel): A small piece of code that "listens" to what your app is doing
  3. The Collector: The pipe that carries telemetry data
  4. The Viewer (Phoenix): The dashboard where you watch the traces

WHY THIS MATTERS FOR FREELANCERS

Clients pay premium rates for AI projects that are well-monitored and debuggable. Adding observability to your projects shows professionalism and makes debugging 10x faster when things go wrong in production.

Step 1: Start Your Dashboard (Arize Phoenix)

Before your app can send data anywhere, you need a place for it to land. The easiest way is to run Phoenix locally on your machine.

In your terminal, run:

pip install arize-phoenix
phoenix serve

What just happened? You started a local server. Open http://localhost:6006 in your browser to see your (currently empty) dashboard.

Step 2: Install the "Flight Recorder"

You need to install the OpenTelemetry (OTel) libraries that specifically understand AI and LLMs. Install the arize-phoenix-otel helper package.

In your project folder, run:

pip install arize-phoenix-otel openinference-instrumentation-openai

Note: If you use LangChain or LlamaIndex instead of raw OpenAI, there are specific "instrumentors" for those too (e.g., openinference-instrumentation-langchain).

Step 3: Turn on the Auto-Recorder

Now, tell your code: "Every time I talk to an LLM, record it and send it to Phoenix." You can do this with just two lines of code at the top of your main script: 

from phoenix.otel import register

# This sets up the plumbing and points it to your local Phoenix server
tracer_provider = register(
    project_name="my-ai-project",
    auto_instrument=True
)

By setting auto_instrument=True, you don't have to manually write "Start Recording" and "Stop Recording" around every function. The OTel library wraps your LLM calls automatically.

Step 4: Run Your Code and Watch

Now run your script. Perform an LLM action (like generating text or classifying data), then go back to http://localhost:6006. You will see a "Trace" appear with complete details about:

  • The exact prompt you sent to the model
  • The response received
  • Latency (how long it took)
  • Token usage and costs
  • Any errors that occurred

Practical Use Cases for Freelancers

Project Type What to Monitor
AI Chatbots Response latency, conversation context window, hallucination detection
RAG Applications Retrieval relevance scores, embedding quality, chunk selection
Content Generation Token costs, generation quality, prompt effectiveness
Data Classification Input/output pairs, accuracy rates, edge cases

Complete Example: Monitoring an OpenAI Call

Here's a full working example combining all the steps:

# 1. Setup observability (do this ONCE at the top of your app)
from phoenix.otel import register

tracer_provider = register(
    project_name="freelance-ai-project",
    auto_instrument=True
)

# 2. Your regular application code
from openai import OpenAI

client = OpenAI()

def classify_content(text):
    """Classify text content - all calls are automatically traced."""
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": "Classify the topic of this text."},
            {"role": "user", "content": text}
        ]
    )
    return response.choices[0].message.content

# 3. Use your function as normal
result = classify_content("AI is transforming freelancing...")
print(result)

# 4. Check http://localhost:6006 to see the full trace!

Moving to Production

When you're ready to move from your laptop to a "real" setup, you just change the Endpoint URL in your code to point to a cloud version of Phoenix. The code itself stays exactly the same: 

# Local development
tracer_provider = register(
    project_name="my-project",
    auto_instrument=True
)

# Production (just add the endpoint)
tracer_provider = register(
    project_name="my-project",
    auto_instrument=True,
    endpoint="https://your-phoenix-cloud-instance.com/v1/traces"
)

PRO TIP

Add observability from Day 1 of your freelance projects. It takes 2 minutes to set up and saves hours of debugging later. Clients are impressed when you can show them real-time dashboards of how their AI system performs.

Summary: Quick Setup Checklist

  1. Install Phoenix: pip install arize-phoenix
  2. Start Dashboard: phoenix serve
  3. Install OTel: pip install arize-phoenix-otel openinference-instrumentation-openai
  4. Add 2 lines: register(project_name="...", auto_instrument=True)
  5. Run & monitor: Open localhost:6006
